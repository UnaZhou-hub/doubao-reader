### 1.1 等级划分

| 等级 | 奥特曼 | 认字量要求 | 等级称号 |
|------|--------|-----------|----------|

- 200 字以下：尚未解锁奥特曼形态，显示「光之国学员」

### 4.1 新增存储字段

```javascript
profile: {
  // 现有字段
  name: '豆包',
  birthday: '2021-06-22',
  
  // 奥特曼等级
  currentLevel: 'ginga',      // 当前奥特曼：geed/ginga/zero/ace/father
  levelUnlockedAt: null,      // 当前等级解锁时间
  
  // 卡片收集
  collectedCards: [
    {
      id: "skill_geed_1",         // 卡片 ID
      count: 2,                    // 获得次数
      firstObtainedAt: "2024-01-10", // 首次获得日期
      lastObtainedAt: "2024-01-15"   // 最近获得日期
    }
  ],
  lastCardDate: "2024-01-15",    // 上次获得卡片的日期（测试奖励）
  
  // 统计数据（用于徽章解锁）
  totalTests: 0,                  // 累计测试次数
  perfectCount: 0,                // 累计 100% 正确次数
  streakDays: 0,                  // 连续学习天数
  lastStudyDate: null,            // 最后学习日期
  totalCardsCollected: 0,         // 累计收集卡片数量（含重复）
  
  // 识字量奖励进度（避免重复领取）
  wordMilestones: [200, 300, 400], // 已领取奖励的识字量里程碑

  // 徽章
  unlockedBadges: [
    { id: 'badge_1', unlockedAt: '2024-01-10' }
  ]

```

### 4.2 旧数据迁移策略

- 从云端加载旧 `profile` 后，直接用新字段默认值覆盖缺失字段（`Storage.init()` 中 spread 合并即可）
- `currentLevel` 不沿用旧值，始终根据当前识字量重新计算：

```javascript
// 旧等级字段（迪迦/赛罗/梦比优斯/贝利亚/奥特之王）直接丢弃
// 初始化时调用 getUltramanLevel() 重新计算
profile.currentLevel = getUltramanLevel(wordBank.length)
```

- 旧 `profile` 中不存在的新字段（`collectedCards`、`wordMilestones` 等）初始化为空值，不报错

### 5.2 卡片设计

- **尺寸**：3:4 竖版（如 300x400 px）
- **格式**：SVG+CSS 动态拼装，不为每张卡片单独绘制图标

#### 分层模板方案

卡片由三层叠加生成，用 JS 动态组合：

```
卡片 = 稀有度底层（边框 + 背景渐变）
     + 奥特曼配色中层（主题色光晕）
     + 类型图标层（通用符号）
     + 文字层（卡片名 + 描述 + 稀有度星标）
```

**第一层 — 稀有度（4套）**：决定边框颜色和背景渐变，见 5.3 节。

**第二层 — 奥特曼主题色（5套）**：

| 奥特曼 | 主色 | 副色 | 特征元素 |
|--------|------|------|---------|
| 捷德   | `#E53E3E` | `#1A1A2E` | 圆形头镖 ○ |
| 银河   | `#C0C0C0` | `#1E3A8A` | 尖角 △ |
| 赛罗   | `#2563EB` | `#1E1B4B` | 双头镖 ◇◇ |
| 艾斯   | `#DC2626` | `#1E3A5F` | L型臂 ⌐ |

## 六、业务逻辑

### 6.1 连续学习定义

- **学习行为**：添加字 OR 完成测试（任意一个即算）
- **计算方式**：按自然日计算，每日至少完成一次学习行为
- **断签处理**：中断后重置为 0

### 6.2 卡片获取边界情况

- 当天第一次测试 0%，第二次 100% → ✅ 可获得卡片
- 当天已获卡片，再次 100% → 提示「今日卡片已获取，明天再来！」
- 识字量达到里程碑（如 300 字）→ 立即弹出卡片，不受每日限制
- 识字量里程碑与升级同时触发时（如第 200 字）→ 先播升级动画并跳转成就页，跳转完成后再弹发卡弹窗

### 6.3 等级更新逻辑

- 每次添加字后检查是否达到下一等级阈值
- 升级时触发庆祝动画（全屏12条光芒射线 + 豆包头像浮动放大 + 等级角标）
- 动画结束后自动跳转到成就页
