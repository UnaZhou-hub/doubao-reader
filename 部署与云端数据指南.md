# 网页部署与云端数据指南

## 概览

- 前端：纯静态 HTML/CSS/JS
- 部署平台：Netlify（免费）
- 云端数据库：Supabase（免费）
- 代码托管：GitHub

---

## 一、GitHub 代码托管

1. 注册 [github.com](https://github.com)，新建仓库（Repository）
2. 将本地代码推送到仓库：
   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/仓库名.git
   git push -u origin main
   ```
3. 后续每次修改代码后：
   ```bash
   git add 修改的文件
   git commit -m "说明改了什么"
   git push
   ```

---

## 二、Netlify 部署

### 首次部署（连接 GitHub）

1. 注册并登录 [netlify.com](https://netlify.com)
2. 点击 **Add new site → Import an existing project**
3. 选择 **GitHub**，授权后找到你的仓库
4. 配置：
   - Build command：**留空**
   - Publish directory：`.`（一个英文点，表示根目录）
5. 点击 **Deploy site**

部署完成后 Netlify 会分配一个域名，如 `xxx.netlify.app`。

### 后续更新

每次 `git push` 到 GitHub 后，Netlify 自动重新部署，无需手动操作。

### 费用与限制

- 静态网站托管永久免费
- 带宽 100GB/月，构建 500次/月（个人项目完全够用）

## 三、Supabase 云端数据库

### 1. 创建项目

1. 注册并登录 [supabase.com](https://supabase.com)
2. 点击 **New project**，填写项目名称和数据库密码

### 2. 建表

进入项目 → 左侧 **SQL Editor → New query**，执行：

```sql
create table doubao_data (
  id uuid default gen_random_uuid() primary key,
  device_id text not null unique,
  word_bank jsonb default '[]',
  records jsonb default '{}',
  profile jsonb default '{}',
  settings jsonb default '{}',
  updated_at timestamptz default now()
);
```

### 3. 开启行级安全（RLS）

在 SQL Editor 中执行：

```sql
alter table doubao_data enable row level security;

create policy "allow read"   on doubao_data for select using (true);
create policy "allow insert" on doubao_data for insert with check (true);
create policy "allow update" on doubao_data for update using (true) with check (true);
```

### 4. 获取 API 密钥

进入项目 → **Settings → API**，复制：
- **Project URL**：`https://xxxxxx.supabase.co`
- **anon public key**：一长串字符

### 5. 在网页中集成

**index.html** — 在 `<head>` 中加入（放在其他 script 之前）：

```html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
```

**js/storage.js** — 顶部加入：

```js
const SUPABASE_URL = '你的 Project URL'
const SUPABASE_KEY = '你的 anon key'
const { createClient } = supabase
const supabaseClient = createClient(SUPABASE_URL, SUPABASE_KEY)

const FAMILY_ID = 'doubao-family'  // 所有设备共用同一个 ID，实现数据同步

async function syncToCloud(data) {
    const { error } = await supabaseClient.from('doubao_data').upsert({
        device_id: FAMILY_ID,
        word_bank: data.wordBank,
        records: data.records,
        profile: data.profile,
        settings: data.settings,
        updated_at: new Date().toISOString()
    }, { onConflict: 'device_id' })
    if (error) console.error('云同步失败:', error)
    else console.log('已同步到云端')
}

async function loadFromCloud() {
    const { data, error } = await supabaseClient
        .from('doubao_data')
        .select('*')
        .eq('device_id', FAMILY_ID)
        .single()
    if (error || !data) return null
    return {
        wordBank: data.word_bank || [],
        records: data.records || {},
        profile: data.profile || {},
        settings: data.settings || {}
    }
}
```

在 `Storage.init()` 中调用 `loadFromCloud()`，在 `saveAll()` 中调用 `syncToCloud()`。

### 6. 验证是否生效

打开网页 → F12 → Console，操作数据后看到 `已同步到云端` 即成功。

### 费用与限制

| 项目 | 限制 |
|---|---|
| 数据库存储 | 500MB（免费） |
| 带宽 | 5GB/月 |
| ⚠️ 注意 | 项目超过 **7 天不活跃**会被暂停，登录 Dashboard 可手动恢复，数据不丢失 |

## 四、多设备数据同步原理

所有设备使用同一个固定的 `FAMILY_ID`（如 `'doubao-family'`），读写 Supabase 中同一行数据，实现跨设备实时同步。
